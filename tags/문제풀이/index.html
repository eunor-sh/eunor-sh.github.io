🚀 3줄 요약 (TL;DR)

  1. 멀티 컨테이너 패턴 구현: emptyDir 공유 볼륨을 활용하여 생성(mc-pod-2)과 출력(mc-pod-3) 역할을 분리한 사이드카 구조의 파드를 배포했습니다.

  2. CRI 환경 최적화: node01 환경에서 cri-dockerd 패키지를 설치하고, 시스템 재부팅 시에도 런타임이 자동 실행되도록 서비스 활성화(Enable) 설정을 완료했습니다.

  3. 서비스 객체 노출 및 검증: 레이블 셀렉터(tier=msg)를 이용해 ClusterIP 타입의 서비스를 생성하고, 엔드포인트 IP와 파드 실제 IP의 일치 여부를 통해 네트워크 연결성을 검증했습니다.
